grep — 文本搜索利器

💡 一句话结论
grep（Global Regular Expression Print）是文本搜索利器。grep pattern file 搜索匹配行。核心参数：-i 忽略大小写 / -v 反向匹配 / -r 递归 / -n 显示行号 / -A/B/C 上下文行 / -E 扩展正则 / -c 计数。常与管道组合。
🎬 生活类比 — grep = 海底捞针的磁铁
用「寻宝」看懂 grep 全部参数
🧲
场景1：grep = 一把能识别材质的磁铁
你在一个装满杂物的巨型仓库里找所有铁制品。grep "pattern" file 就是那把磁铁——它扫过每一排货架（每行），吸出所有含铁的东西（匹配行）。仓库太大？grep -r 递归搜索 = 每个抽屉都翻一遍。
🎨
场景2：-i = 不分颜色，-v = 反过来找
grep -i "GOLD" = 不管金色银色铜色，只要是金属都算（忽略大小写）。grep -v "铁" = 反过来——把所有不含铁的杂物找出来（反向匹配）。清理仓库时，先把不是铁的拣出来扔掉。
📍
场景3：-n = 给你坐标，-C = 连邻居一起看
grep -n = 不仅告诉你找到了，还告诉你在第几排第几个货架（行号）。grep -C 5 = 把找到的物品前后 5 个邻居也一起展示——排查问题时，上下文比物品本身更重要。

📋 7步答题模板

第一步：一句话定义

grep 是 Linux 三剑客之一，用于文本搜索和过滤。它能用正则表达式在文件中查找匹配行，并输出结果。名称来自 ed 编辑器命令 g/re/p。

第二步：基础语法

grep [options] pattern [file...]。最简单的用法：grep "error" app.log。pattern 支持正则表达式（默认 BRE），-E 启用扩展正则（ERE），-P 启用 Perl 正则。

第三步：核心六参数

-i（ignore case 忽略大小写）、-v（invert 反向匹配，输出不匹配的行）、-r（recursive 递归搜索目录）、-n（line number 显示行号）、-c（count 只显示匹配行数）、-o（only-matching 只输出匹配部分而非整行）。

第四步：上下文查看

-A N（After，匹配行之后 N 行）、-B N（Before，匹配行之前 N 行）、-C N（Context，前后各 N 行）。日志排查时的标配：grep -C 5 "error" app.log。

第五步：管道组合（实战核心）

grep 的价值在管道中放大。经典组合：ps aux | grep nginx（查看 nginx 进程）、tail -f app.log | grep ERROR（实时过滤错误）、cat access.log | grep 404 | awk '{print $1}'（找 404 的来源 IP）。

第六步：正则技巧

grep -E "error|fail|panic" log（或关系）、grep "^2024-" log（匹配行首）、grep "timeout$" log（匹配行尾）、grep -E "[0-9]+ms" log（匹配数字+ms）。

第七步：性能与进阶

大数据用 fgrep（grep -F）做固定字符串匹配，不用正则更快。grep --exclude="*.log" pattern * 排除特定文件。大文件配合 head/tail 减少扫描量。
