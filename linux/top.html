top — Linux 实时系统监控工具

💡 一句话结论
top 是 Linux 实时系统监控工具，核心指标：load average（1/5/15 分钟平均负载，多核 CPU 不超过核心数）、CPU 使用率（us/sy/id/wa）、内存使用（free/used/buff/cache）、进程状态（R/S/D/Z）。
🎬 生活类比 — top = 汽车仪表盘
用「开车」看懂 top 全部指标
🏎️
场景1：load average = 发动机转速表
你开车上高速，转速表指针飙到 8000 转——发动机在嘶吼（load average 飙升）。4 缸发动机（4 核 CPU）正常巡航 2000 转，长期超 4000 转？引擎要炸（系统过载）。load average 是综合拥堵指数，1min/5min/15min = 刚上路/跑一会/巡航期的转速。
⛽
场景2：CPU = 油门，内存 = 油箱
油门踩多深 = us+sy（CPU使用率），踩到底就是 100%。油箱还剩多少 = free+buff/cache（可用内存）——油表不准因为有些油在管路里（buff/cache 可回收）。wa 高 = 等红灯时发动机空转（CPU 在等 IO 没法干活）。
🧑‍🤝‍🧑
场景3：进程状态 = 乘客们
R 状态 = 正在飙车的车手（运行中）。S 状态 = 副驾睡着了（可中断睡眠）。D 状态 = 昏迷的乘客，谁也叫不醒（不可中断，等 IO）。Z 状态 = 已经去世的乘客（僵尸进程，父进程没来收尸）。Swap 用了 = 后备箱塞满了，开始往车顶绑行李。

📋 7步答题模板

第一步：一句话定义

top 是 Linux 下最常用的实时系统监控工具，类似 Windows 的任务管理器。它动态显示系统资源使用情况和进程列表。

第二步：核心输出区域

top 输出分两部分：上半部分是系统统计信息（System Summary），下半部分是进程列表（Task List）。

第三步：load average（必考点）

load average 是系统 1分钟/5分钟/15分钟的平均负载，表示处于 R 状态（运行）和 D 状态（不可中断睡眠）的进程数。判断标准：单核 ≤1，多核 ≤CPU 核心数。例如 4 核 CPU，load average 长期超过 4 说明系统过载。

第四步：CPU 行解读

us（用户态 CPU）/ sy（内核态 CPU）/ ni（低优先级用户态）/ id（空闲）/ wa（等待 IO 的 CPU %）/ hi（硬中断）/ si（软中断）/ st（被虚拟机偷走）。重点关注 us+sy（CPU 使用率）和 wa（IO 等待）。wa 持续很高 → 磁盘 IO 瓶颈。

第五步：内存行解读

Mem：total/free/used/buff/cache。注意 Linux 会把空闲内存做缓存，所以 free + buff/cache ≈ 实际可用内存。Swap 行显示交换分区使用，swap used 持续增长 = 内存不足在换页。

第六步：进程状态

R（Running 运行中）、S（Sleeping 可中断睡眠）、D（Disk sleep 不可中断睡眠，等待 IO）、Z（Zombie 僵尸进程，父进程未回收）、T（Stopped 暂停）。Z 状态进程不占用资源但占 PID，过多说明父进程有 bug。

第七步：常用快捷键

P - 按 CPU 使用率排序、M - 按内存使用排序、1 - 展开所有 CPU 核心、c - 显示完整命令、k - 杀死进程、q - 退出、t/m - 切换 CPU/内存显示样式、H - 切换线程视图。

⏱️
60秒面试表达
"top 是 Linux 实时系统监控的标配工具。我最关注三个指标：一是 load average，衡量系统整体繁忙程度，多核 CPU 不超过核心数；二是 CPU 行的 us+sy 和 wa，分别反映计算和 IO 压力；三是内存的 buff/cache 要算进可用内存。进程状态 R/S/D/Z 也很有用，特别是 D 状态代表进程在等待 IO 无法被打断，Z 代表僵尸进程需要父进程回收。"

⚠️ 面试常见错误 vs 正确回答

❌ 错误回答

"load average 就是 CPU 使用率"

"free 内存太少系统不行了"

"Z 进程要立刻 kill 掉"

✅ 正确回答

"load average 是运行队列+等待 IO 的任务数，不是 CPU 使用率"

"free 少 ≠ 内存不足，buff/cache 是可以回收的可用内存"

"Z 进程已经死了，kill 没用，应该找父进程修复 wait()"

🌲 追问树

top
├─ load average 是什么？
→ 运行(R)+不可中断睡眠(D)的进程数均值
├─ 多核怎么判断？
→ load ≤ CPU 核心数，可用 nproc/lscpu 查看
├─ CPU 行各字段含义？
→ us 用户 / sy 系统 / id 空闲 / wa IO等待
├─ 内存行怎么看真正剩余？
→ free + buff/cache = 可用内存（available）
├─ 进程状态 R / S / D / Z / T 分别是什么？
→ R 运行、S 可中断睡眠、D 不可中断(IO)、Z 僵尸、T 暂停
└─ 快捷键有哪些？
→ P(按CPU) M(按内存) 1(展开CPU) c(完整命令) k(kill)
