tail — 实时日志追踪与排查

编号：LINUX-07难度：⭐⭐频率：🔥🔥🔥🔥🔥
💡 一句话结论
tail -f file 实时追踪文件新增内容，tail -n 100 查看最后 100 行。日志排查三板斧：tail -f 追踪 + Ctrl+C 停止、grep 过滤、awk 提取列。配合 tee 同时显示和写入。日志切割用 logrotate 避免文件过大。
🎬 生活类比 — tail -f = 追剧
用「追剧」看懂日志排查
📺
场景1：tail -f = 追剧模式，只看最新更新
你追的电视剧更新了。tail -f app.log = 打开追剧模式——屏幕一直播，来一集看一集（实时看日志新增内容）。不想看了？Ctrl+C = 关电视。想要"边看边录"？tail -f log | tee -a save.log = 一边看一边录下来。
🎬
场景2：head = 看预告片，grep = 只看特定演员
head -n 20 = 看预告片（文件前 20 行），先了解讲了什么。tail -f log | grep ERROR = 追剧时只看有特定演员（ERROR）出场的集数。排查问题时，先 grep 定位角色，再用 -C 5 看他出场前后五分钟的剧情（上下文）。
🔄
场景3：tail -F = 追剧时换源自动续播
追的剧突然完结（日志被切割轮转），普通 tail -f 就停了（傻傻等旧文件）。tail -F 更聪明——发现剧换了新季（日志切割成新文件），自动追过去继续看。logrotate = 每季规定集数到了就换新剧集（按大小/时间切割）。

📋 7步答题模板

第一步：一句话定义

tail 是查看文件末尾内容的命令，-f（follow）模式是实时日志排查的核心工具，会持续输出文件新增的内容。

第二步：核心参数

tail -n N：显示最后 N 行（默认 10 行）。tail -f：持续追踪文件新增内容（ctrl+c 退出）。tail -F：同 -f 但文件被删除/轮转时会自动重新打开（更推荐）。tail -c BYTES：按字节数显示末尾。

第三步：日志排查三板斧

①tail -f /var/log/app.log | grep ERROR（实时过滤错误）；②ctrl+c 停止后，用 grep 关键词搜索上下文；③awk 提取关键列做统计分析。这是线上排查的标准流程。

第四步：head 补充

head -n N：查看文件前 N 行（默认 10）。head -c BYTES：按字节。tail + head 组合可以提取文件任意片段。tail -n +100 file | head -n 50（从第 100 行开始取 50 行）。

第五步：tee 双输出

tail -f app.log | tee output.log：一边看着屏幕，一边写入文件。tail -f app.log | tee -a output.log：追加写入。也可管道给 grep：tail -f app.log | tee -a output.log | grep ERROR。

第六步：日志切割 logrotate

单文件日志过大？logrotate 定期切割（rotate）：按大小/时间、压缩（compress）、保留份数（rotate N）、执行脚本（postrotate）。tail -F 能自动跟踪切割后的新文件（-f 不会）。

第七步：journalctl（SystemD 时代）

现在很多系统用 SystemD 管理服务，日志走 journald。journalctl -u service-name -f 实时查看服务日志。journalctl -u service-name --since "10 min ago" 按时间过滤。journalctl -u service-name -n 100 最后 100 行。

60秒面试表达
"tail -f 是最常用的日志排查命令。线上出了问题，我的标准流程：先 tail -f app.log | grep ERROR 看实时错误，定位到关键行后 ctrl+c，用 grep -C 看上下文，再用 awk 提取关键数值统计。tail -F 比 -f 更好——日志切割后能自动跟踪新文件。另外 cut 只能简单切列，awk 能做聚合计算，sed 做原地替换。配合 tee 可以边看边存。"
