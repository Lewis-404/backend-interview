awk — 文本数据处理语言

编号：LINUX-04难度：⭐⭐⭐频率：🔥🔥🔥🔥
💡 一句话结论
awk 是文本数据处理语言，按行读取并按列（$1/$2/$NF）处理，支持条件过滤、计算、格式化输出。经典用法：awk '{print $1,$3}'、awk '$3>100'、awk '{sum+=$2}END{print sum}'。
🎬 生活类比 — awk = Excel 命令行版
用「Excel 表格」看懂 awk 数据处理
📊
场景1：$1 = A列，$2 = B列
你打开一个 Excel 表格。$0 是整行（一整行数据）、$1 是 A 列、$2 是 B 列…$NF 是最后一列。-F 就是指定分隔符——默认是空格/Tab，-F: 就是用冒号切列（像 Excel 的"分列"功能）。
🔢
场景2：条件过滤 = 筛选，计算 = SUM
awk '$3>100' = Excel 里对第三列筛选"大于 100"。awk '{sum+=$2}END{print sum}' = 把 B 列全部加起来，最后输出总和——这就是 Excel 的 SUM。awk '{arr[$1]++}END{for(k in arr) print k,arr[k]}' = Excel 的数据透视表！
📝
场景3：BEGIN = 页眉，END = 页脚
BEGIN { print "姓名,成绩" } = 打印表头（第一行前执行）。END { print "平均:", total/NR } = 打印汇总行（所有数据处理完后执行）。NR = 当前行号（第几行），NF = 当前行列数（一共几列）。

📋 7步答题模板

第一步：一句话定义

awk 是 Linux 三剑客中最强大的文本处理工具，它是一个完整的文本数据处理语言。按行读取，默认按空格/Tab 分割列，支持计算、过滤、格式化输出。

第二步：基本结构

awk 'pattern { action }' file。pattern 是条件（可选），action 是处理动作。不写 pattern 则每行都处理；不写 action 则打印匹配行（默认 {print}）。

第三步：列操作（核心）

$0 = 整行、$1 = 第一列、$2 = 第二列、$NF = 最后一列、$(NF-1) = 倒数第二列。可以指定分隔符：-F: 用冒号分割（如 /etc/passwd）、-F"," 用逗号分割（CSV）。

第四步：内置变量

NR（行号，从 1 开始）、NF（当前行的列数）、FS（输入分隔符，默认空格）、OFS（输出分隔符，默认空格）、RS（行分隔符，默认换行）、ORS（输出行分隔符）。awk 'NR==1 || NR==10' 打印第1行和第10行。

第五步：BEGIN 和 END 块

BEGIN 块在处理第一行之前执行（初始化变量、打印表头）；END 块在处理完所有行之后执行（打印汇总结果）。awk 'BEGIN{print "开始"} {sum+=$2} END{print "合计:",sum}'。

第六步：条件过滤与计算

条件过滤：awk '$3 > 100' file、awk '$1 ~ /error/' file（正则匹配）。统计计算：awk '{sum+=$2; count++} END{print sum/count}'（求第二列平均值）。awk '{arr[$1]++} END{for(k in arr) print k,arr[k]}'（分组计数）。

第七步：实战场景

日志分析：awk '{print $1}' access.log | sort | uniq -c | sort -nr（统计访问 IP 排行）。列提取：ps aux | awk '$3>50 {print $2,$11}'（CPU>50% 的进程）。格式转换：awk '{print $2","$3}' 转 CSV。

60秒面试表达

⚠️ 面试常见错误 vs 正确回答

❌ 错误回答

"awk 只能打印列"

"awk 和 cut 差不多"

"awk 只能处理格式化文本"

✅ 正确回答

"awk 有完整的条件判断、循环、变量、数组，是一种编程语言"

"cut 只能简单切列，awk 能做计算/过滤/聚合/格式化"

"awk 可以处理任意分隔符的文本（-F），也能做模式匹配和正则"
