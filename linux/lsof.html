Linux / lsof 文件描述符

lsof — 列出打开的文件（一切皆文件）

编号：LINUX-06难度：⭐⭐⭐频率：🔥🔥🔥🔥
💡 一句话结论
lsof（List Open Files）查看进程打开的文件/端口/连接。lsof -p PID 查看指定进程，lsof -i:PORT 查看端口占用，lsof -i TCP 查看网络连接，lsof /path/to/file 查看谁在用该文件。一切皆文件思想。
🎬 生活类比 — lsof = 私家侦探
用「侦探破案」看懂 lsof 三大场景
🕵️
场景1：查人 — lsof -p PID = 跟踪嫌疑人
侦探接到案子：这个进程（嫌疑人）到底在干嘛？lsof -p PID 列出他打开的所有东西——正在看什么文件、和谁聊天（socket）、用了什么工具（设备）。嫌疑人打开太多文件？FD 泄漏——就像一个人同时和 100 个人聊天，迟早崩溃（Too many open files）。
🚪
场景2：查门 — lsof -i:PORT = 查谁开了这扇门
大厦保安发现 8080 号房间有人进出。lsof -i:8080 = 调监控看谁在用这个房间（端口）。"端口被占用无法启动服务"？侦探一秒破案——找出占着房间的人（进程），请他搬走（kill）。lsof -i TCP = 调全部网络监控。
📁
场景3：查物 — lsof /path/to/file = 追踪失窃物品
你删了一个大文件但磁盘空间没释放。lsof /path/to/file = 侦探告诉你：虽然登记册上划掉了（目录删除），但有进程还在抓着它不放！只有让那个进程放手（重启），空间才能真正释放。这就是 Linux "一切皆文件" 的体现。

📋 7步答题模板

第一步：一句话定义

lsof 是排查问题和诊断的利器，体现 Linux "一切皆文件" 的哲学。它列出当前系统所有打开的文件描述符（File Descriptor），包括普通文件、目录、网络 socket、管道、设备文件等。

第二步：核心用法

lsof（不加参数）= 列出所有打开的文件（非常多）。lsof -p PID = 查看指定进程打开的所有文件。lsof -u USER = 查看某用户打开的文件。lsof -c CMD = 查看以指定命令开头的进程。

第三步：网络诊断利器

lsof -i:PORT = 查看哪个进程占用了该端口（最常用）。lsof -i TCP = 列出所有 TCP 连接。lsof -i UDP = 列出所有 UDP 连接。lsof -i @HOST = 列出到指定主机的连接。lsof -i -sTCP:LISTEN = 只显示监听的 TCP 端口。

第四步：文件锁排查

lsof /path/to/file = 查看哪个进程在使用该文件。删除文件但磁盘没释放？lsof 找还在 hold 文件的进程。lsof /var/log/app.log → 如果日志被删但进程还在写，需要用 lsof 找到进程并重启。

第五步：文件描述符泄漏诊断

lsof -p PID | wc -l = 统计某进程打开的文件数。FD（文件描述符）列显示：cwd（当前目录）、rtd（根目录）、txt（可执行文件）、mem（内存映射文件）、数字+字母（实际 fd）。打开文件数持续增长 → 可能有 fd 泄漏。

第六步：fd 上限（ulimit -n）

每个进程有文件描述符数量限制：ulimit -n 查看软限制，-Hn 查看硬限制。Too many open files 错误 → 调大 ulimit -n 或修复 fd 泄漏。Linux 默认 1024，生产环境通常设 65535 或更高。

第七步：与 /proc/PID/fd 关系

/proc/PID/fd/ 目录下有每个打开的文件描述符的符号链接（以 fd 编号命名）。lsof 本质就是遍历这些信息并格式化输出。可以直接 ls -l /proc/PID/fd 查看。

⚠️ 面试常见错误 vs 正确回答

❌ 错误回答

"lsof 就是查端口的"

"文件删了就释放磁盘了"

"too many open files 只能重启"

✅ 正确回答

"lsof 能查文件、端口、socket、管道、设备——一切皆文件"

"文件删了但进程还持有 fd 时磁盘不会释放，lsof 找到进程重启才行"

"too many open files → 调 ulimit -n 提高限制，同时排查是否有 fd 泄漏"

🌲 追问树

lsof
├─ lsof -p PID 看到什么？
→ 进程打开的所有 FD：文件、socket、管道、设备、内存映射
├─ lsof -i:PORT 查什么？
→ 端口被哪个进程占用，最常用的端口排查命令
├─ lsof -i TCP 显示什么？
→ 所有 TCP 连接，含状态和远端地址
├─ lsof -u USER 怎么用？
→ 查看某个用户打开的所有文件
├─ FD 泄漏怎么诊断？
→ lsof -p PID | wc -l 看趋势，持续增长就是泄漏
└─ fd 上限是多少？
→ ulimit -n 查看 / -Hn 硬限制，默认 1024，生产设为 65535+
