💡 一句话结论
Shell 脚本是 Linux 自动化核心：变量（赋值无空格 variable=value）、条件（[ ] / [[ ]] / (( ))）、循环（for/while）、函数、错误处理（set -e / set -u）、参数传递（$1 / $@ / $#）。shebang：#! /bin/bash。

📋 7步答题模板

第一步：一句话定义

Shell 脚本是 Linux 系统管理的基石，用于自动化执行命令序列。常见 Shell 有 bash（默认）、zsh、sh。第一行 shebang #! /bin/bash 指定解释器。

第二步：变量（必考点）

赋值不能有空格：name="hello"（正确）、name = "hello"（错误！）。引用用 $name 或 ${name}（后者可拼接）。环境变量：export VAR=value 让子进程继承。只读变量：readonly VAR。常用特殊变量：$0 脚本名、$1-$9 位置参数、$@ 所有参数（分开）、$* 所有参数（整体）、$# 参数个数、$? 上条命令退出码、$$ 当前 shell PID。

第三步：条件判断

test / [ ]：字符串比较 [ "$a" = "$b" ]、数字比较 [ $a -eq $b ]、文件判断 [ -f file ] [ -d dir ] [ -x file ]。[[ ]]：bash 增强版，支持 && || 和正则 =~，变量不用加引号也不会 word splitting。(( ))：算术运算 ((a+b>10))。

第四步：循环

for i in list; do ... done（遍历列表）。for ((i=0; i<10; i++)); do ... done（C 风格）。while condition; do ... done（条件循环）。break 跳出、continue 跳过当前。循环读文件：while read line; do echo $line; done < file.txt。

第五步：函数

定义：function name { ... } 或 name() { ... }。调用：name arg1 arg2（不需要括号）。参数：$1 $2 在函数内部也是位置参数。返回值：return 只能 0-255（成功/失败），数据通过 echo 返回：result=$(my_func)。

第六步：错误处理

set -e：任何命令失败立即退出（推荐）。set -u：使用未定义变量时报错。set -x：调试模式，打印每条执行的命令。set -o pipefail：管道中任一命令失败则整个管道失败。trap：捕获信号 → trap 'echo cleanup; exit' EXIT INT TERM。

第七步：实战脚本模板

#!/bin/bash、set -euo pipefail、变量定义、参数检查（if [ $# -lt 1 ]; then echo "Usage..."; exit 1; fi）、主逻辑、错误处理 trap。典型场景：服务启动脚本、日志清理、批量部署、定时备份。

⏱️
60秒面试表达
"Shell 脚本是运维自动化标配。关键语法：变量赋值不能有空格、[[]] 比 [] 更安全支持正则、$? 判断上条命令成败、${var:-default} 给默认值。错误处理必须做——set -euo pipefail 三件套 + trap 清理。$@ 和 $* 的区别也常被问到：$@ 是分开的参数 "$1" "$2"，$* 是整体 "$1 $2"。"

⚠️ 面试常见错误 vs 正确回答

❌ 错误回答

"name = value 就行"

"[ ] 和 [[ ]] 一样"

"Shell 脚本不需要错误处理，跑完就行"

✅ 正确回答

"赋值绝对不能有空格：name=value，等号两边紧贴"

"[[ ]] 是 bash 增强版，支持 && || 和正则，变量不需要引号保护"

"set -e + trap 是生产脚本的底线，不做的脚本迟早出生产事故"

🌲 追问树

Shell 脚本
├─ 变量怎么定义和使用？
→ name=value（无空格）、$name、${name}、export、readonly
├─ [ ] vs [[ ]] vs (( )) 有什么区别？
→ [ ] = POSIX test、[[ ]] = bash 增强(正则/&&/||)、(( )) = 算术
├─ 循环怎么写？
→ for i in list / for((;;)) / while、break/continue
├─ 函数怎么定义和调用？
→ name(){...}、调用 name arg1 arg2、return 0-255、echo 返回数据
├─ 错误处理怎么做？
→ set -e(失败退出) / -u(未定义报错) / -x(调试) / trap 信号捕获
└─ 特殊参数有哪些？
→ $1-$9(位置) / $@$* / $#(个数) / $?(退出码) / $$(PID) / $!(后台PID)
